# 关于CSS中margin的百分比设置

设置 margin 的百分比的时候，注意四个方向上的百分比计算的结果都是以 width 为基准计算得到的，因此宽度上可以实现自适应，但是高度上往往会出现问题

# 单位：vh和vw

vh和vw都是css中的相对长度单位，vh表示当前视窗的高度（浏览器可视区域），视窗高度被分为100单位，即1vh永远等于视窗高度的百分之一。（window.innerHight, window.innerWidth）

应用场景：图片的尺寸限制，例如图片过大的时候，一屏显示区域不够，直接做法是获得图片的原始大小，以及浏览器的大小尺寸，然后设置比例进行调整，使用css则可以：

.vw-vh-img{
    max-width: 90vw;
    max-height: 90%;
    max-height: 90vh;
}

# 注意外边框合并的问题

相邻的两个盒子（兄弟或祖先），其外边距可以合并成一个单独的外边距，如果是父子元素，较简单的方式有：

  • 给父元素添加 padding值（增加了padding误差）
  • 父元素或子元素声明浮动或者绝对定位
  • 父元素添加属性： overflow: auto/auto/scroll
  • 将子元素的margin使用父元素的padding代替

# JS获取元素 margin padding等属性值

直接使用 style.margin 无法获取到margin值，element.style. 只能读取到元素的内联样式，即写在元素的style属性中的样式，可以进行读写。

getComputedStyle则是获取元素的最终样式，包括了内联、嵌入以及外部样式，但是不支持写。

综上，可以通过 getComputedStyle 读取样式，然后通过 element.style.进行修改

var divChil = document.querySelector('.chil');
console.log(divChil.offsetHeight);
console.log(divChil.offsetWidth);

var compStyle = getComputedStyle(divChil, null);
console.log(compStyle.marginTop);
console.log(compStyle.marginBottom);
console.log(compStyle.marginLeft);
console.log(compStyle.marginRight);

# 关于 div 的点击事件

页面中存在多个重叠的div，要注意 div 的放置顺序 （最大的div应放置在页面最底层，按照大小依次放置），否则在设置 div 的点击事件时会出现问题

# i 标签使用

经常使用 i 标签实现类似某些特定位置的关闭按钮，通过设置 font-style: normal 设置内容不再倾斜，设置 cursor: pointer 设置鼠标样式

# img标签按原始比例改变大小

对 img 样式进行设置时，尽量只设置 height 或 width 其中的一个，这样图片会自动按照原始比例进行缩放

# 布局、定位、浮动

# 1.display

# 1.1 display: none

元素不会被显示，同样也不会占用空间

# 1.2 display: block

将元素显示为块级元素，前后有换行符。

对于设置此属性的行内元素，默认会继承父元素的宽度，并且即使宽度有剩余也会独占一行（margin撑开），高度如果未设置会以子元素撑开的高度为准，也可以自己设置高度

  • 块级元素如div，p，h1等display的默认值为block
  • 块级元素的高度、宽度等都可以设置
  • 宽度在不设置的i情况西，默认与父元素保持一致

# 1.3 display: inline

将元素显示为行内元素，此时块级元素的高度，宽度，margin等设置都会失效，元素的宽度就是其包含的文字或者图片的宽度

# 1.4 display: inline-block

显示为行内块元素，同时具有行内元素和块级元素的一些特点：

  • 具有高宽属性
  • 可以放置在同一行
  • 注意行内块元素会有小margin，显示会有缝隙

# 2. 浮动

浮动基本的出发点：如何在一行中显示多个div元素，浮动可以让元素脱离标准流，漂浮在标准流之上。

以 div 元素为例，如果某个 div 元素是浮动的，位于其上的元素如果也是浮动的，那么该div会跟随在上一个元素后面（如果一行放不开则会挤到下一行），位于其上的元素如果是标准流的，则该div会与上个元素的底部对齐。

清除浮动可以设置：overflow：hidden

此外，注意对于 li 设置浮动属性的时候，由于脱离了正常的文档流，因此父元素div（如果将ul放在一个div里的话），其高度会不计入这些 li 元素，从而出现 li 高度超出 div 的现象，如果 div 中没有其他元素，div 会 “消失” -- 高度为0。 因此往往其父元素会直接指定 height 属性，假装将浮动的这些元素包括在里面。

总之，浮动的特点：

  • 脱离标准流，不占位置

  • 浮动元素之间默认没有缝隙

  • 浮动后可以直接设置宽度和高度等属性

  • 浮动只会影响后面的标准流盒子，不会对之前的标准流盒子造成影响

  • 浮动压不住父元素的边框以及pading区域（定位可以）

  • 浮动之后不占有位置，父元素如果没有设置height（默认height为0），则父元素检测不到高度

  • 清除浮动：清除加了浮动对后面元素的影响

    • 给父元素添加 overflow：hidden 属性 （可能会隐藏必要内容）

    • 伪元素清除浮动：

        .clearfix::before,
        .clearfix::after {
            /* 要想使用伪元素，必须写上content属性 */
            content: "";
            /* 这里写成block也是没有问题的，为什么使用table,这是老的写法，兼容老版本浏览器 */
            display: table;
        } 
        .clearfix::after {
            clear: both;
        } 
        .clearfix {
            *zoom: 1;
        }

    var ol = document.querySelector('ol');

# iframe 标签

注意<iframe>标签存在的问题：如果一开始设置某个 iframe 为隐藏状态（ display: none 或者设置为 hide），那么浏览器会偶尔忽略元素内容的呈现，造成 iframe 的内容空白

解决方法为：使用定位将 iframe 隐藏到页面可视区域外

此外，引入 bootstrap 也会导致这一问题

# JS 获取数组最大最小值

Math 提供的函数 Math.min() ，Math.max() 使用形式为：

console.log(Math.min(1,2,3))
1

如果检测到其中某个值非数值，则会返回 Nan， 因此无法直接传入数组，如果要对数组进行操作，需要：

console.log(Math.min.apply(Math, arr1))
1

其中 Math.min.apply() 的第一个参数可以使用任意对象
